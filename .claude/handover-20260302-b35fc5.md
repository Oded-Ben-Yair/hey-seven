### The 3 Layers of Agent Quality
```
Layer 1: Infrastructure (R76 fixed)
  - Schemas work, dispatch works, extraction works, crisis persists
  - Score contribution: enables everything, but doesn't improve scores alone

Layer 2: Content (R77-R78 target)
  - RAG content quality, specialist knowledge, prompt specificity

Layer 3: Behavioral (R79-R80 target)
  - Tone calibration, memory usage, engagement adaptation, proactive intelligence
  - Score contribution: +1-2 points per dimension
  - CANNOT work without Layers 1+2
```

### The Fallback Cascade (Root Cause Chain)
```
Guest says "Great, we'll do that" (confirmation)
  → Router classifies as "ambiguous" or "property_qa"
  → RAG retrieval runs (retrieves irrelevant context for confirmations)
  → Specialist generates response using stale/wrong context
  → Validator rejects for grounding violation
  → Retry: specialist tries again, still grounded in wrong context
  → Validator rejects again → FAIL
  → Fallback: "I don't have enough specific details..."

FIX: Intercept confirmations BEFORE RAG retrieval.
```

### The "Explore Rewards" Loop (Root Cause Chain)
```
Guest asks about loyalty/comps/rewards/promotions
  → Router classifies as "property_qa" (correct)
  → RAG retrieves thin promotions/loyalty data
  → Dispatch routes to comp agent (correct)
  → Comp agent generates response from thin RAG context
  → Only info available: "Momentum program exists"
  → Agent says "I'd love to help you explore rewards, tell me more"
  → Guest provides more info → same thin RAG → same loop

FIX: Add real Momentum tier data to knowledge-base. OR change comp agent
to honestly redirect: "For your specific Momentum tier benefits, contact
your casino host at 1-888-226-7711."
```
