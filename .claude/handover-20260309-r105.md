### Priority 1: LangGraph Tool-Use for Comp (H9 → 5.0+)

**Current**: CompStrategy injects policy tables into system prompt. Model ignores them.
**Target**: Agent CALLS a tool mid-conversation to look up comp eligibility. Tool returns a structured response that becomes part of the conversation, not the prompt.

```python
# New: Agent calls comp tool during conversation
from langchain_core.tools import tool

@tool
def check_comp_eligibility(guest_tier: str, occasion: str | None = None) -> str:
    """Check what comps a guest qualifies for based on their tier and occasion."""
    result = get_comp_strategy(CompStrategyInput(
        guest_tier=guest_tier,
        current_occasion=occasion,
    ))
    # Return natural language, not structured data
    if not result.eligible_comps:
        return "No specific comps available right now, but rewards program enrollment is always free."
    offers = [f"- {o.description} (~${o.estimated_value:.0f})" for o in result.eligible_comps[:3]]
    return f"Based on {result.tier_name} tier:\n" + "\n".join(offers)
```

This changes H9 from "model reads comp policy in prompt" to "model decides to check comps and gets results." The decision to check is the key behavioral change.

**Implementation plan**:
1. Create `src/agent/tools/comp_tools.py` — LangGraph-compatible tools
2. Add tool_node to graph (between router and specialist dispatch)
3. Router recognizes comp queries → routes to tool-calling path
4. Tool results become AIMessage content (model sees them as conversation)
5. Test with 10 comp-focused scenarios, then full 30 HT

## Deep Learnings (for Memory MCP + Rule Files)

### 1. Prompt Engineering Has a Quantifiable Ceiling
After R102's identity rewrite, ALL subsequent prompt changes (R103-R105) produced ±0.3 on ALL dimensions. This is statistically within judge variance (GPT-5.2 ±0.09). The ceiling is NOT the prompt — it's the model's ability to follow multi-section prompts during generation.

### 2. Instruction Register Doesn't Matter
"MUST mention this offer" and "if it feels natural, you can mention this offer" produce identical LLM behavior. The model's incentive-mention rate is driven by context salience (is the offer relevant to what the guest is talking about?), not instruction forcefulness.

### 3. Extraction Prompt Wording Doesn't Change Extraction Rate
"Be extremely generous" and "extract what was STATED or STRONGLY IMPLIED" produce the same extraction. The model extracts what it recognizes, not what the prompt asks for. Improvement requires fine-tuning on annotated examples.

### 4. Code Review Catches What Tests Can't
3789 tests passed while handoff prompts were dead code. Tests verify data flow (handoff_request dict is correct), not prompt flow (LLM sees the right context). Need integration tests that assert response CONTENT matches expected prompt influence.

### 5. Parallel Worktree Agents Work at Scale
3 agents in parallel, strict file ownership, code-judge hostile review → +1734 LOC, +252 tests, 4 bugs caught. Pattern validated. Key: define file ownership tables, not just task descriptions.

### 6. The Model's Behavioral Ceiling = Identity Prompt × Few-Shot Examples
R102's "casino host building a relationship" + 27 gold trace examples set the ceiling. Everything after that is noise. The next ceiling-break requires either (a) model fine-tuning to internalize better behavior, or (b) architecture changes that give the model new CAPABILITIES (tools, actions, real data).
