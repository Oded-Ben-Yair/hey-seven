## Rebutted Findings (False Positives)

### 1. RAG Chunk ID Collision (D2, scored as MAJOR)

**Reviewer claim**: "`pipeline.py` hashes `text + source` with no delimiter, while `reranking.py` explicitly documents why you need a delimiter."

**Evidence**: This was fixed in R36 (commit from 2026-02-23). The reviewer missed the fix.

`src/rag/pipeline.py:242-244`:
```python
    # R36 fix A5: Use null byte delimiter to prevent ambiguous concatenation.
    # Without delimiter: text="abc", source="def" == text="abcde", source="f".
    return hashlib.sha256(f"{text}\x00{source}".encode()).hexdigest()
```

`src/rag/reranking.py:57-60`:
```python
            # R36 fix A8: Use null byte delimiter to prevent ambiguous
            # concatenation (same fix as _compute_chunk_id in pipeline.py).
            doc_id = hashlib.sha256(
                f"{doc.page_content}\x00{doc.metadata.get('source', '')}".encode()
            ).hexdigest()
```

Both files use identical `\x00` null-byte delimiter. The shared `_compute_chunk_id()` helper at `pipeline.py:228` is the single source of truth for ingestion IDs.

**Verdict**: False positive. No action needed.

### 3. Circuit Breaker State Mutation Outside Lock (D8, scored as MAJOR)

**Reviewer claim**: "Circuit breaker correctness under concurrency is explicitly suspect because `state` can mutate outside lock."

**Evidence**: The `state` property is **read-only** and documented as monitoring-only. All mutations happen under `asyncio.Lock`.

`src/agent/circuit_breaker.py:239-247` (read-only property):
```python
    @property
    def state(self) -> str:
        """Current circuit breaker state (read-only, no mutation).

        To check whether to allow a request, use ``allow_request()``
        which performs the open -> half_open transition under the lock.
        """
        if self._state == "open" and self._cooldown_expired():
            return "half_open"
        return self._state
```

`src/agent/circuit_breaker.py:253-267` (`is_open` property with explicit warning):
```python
    @property
    def is_open(self) -> bool:
        """Approximate breaker state for monitoring/health checks only.

        **WARNING**: This is a non-atomic read (no lock). ...
        For **control flow** (deciding whether to allow a request), always use
        ``allow_request()`` which performs the state check under ``self._lock``.
        For **monitoring/health checks**, this property is acceptable.
        """
```

All control-flow mutations:
- `allow_request()` — line 368: `async with self._lock:`
- `record_success()` — line 416: `async with self._lock:`
- `record_failure()` — line 484: `async with self._lock:`
- `record_cancellation()` — line 465: `async with self._lock:`
- `_apply_backend_state()` — called inside locked sections only

**Verdict**: False positive. The reviewer confused monitoring properties (intentionally unlocked, documented) with control-flow mutations (all locked).

### 5. Retrieval Pool Hardcoded to 50 Workers (D8)

**Reviewer claim**: "Retrieval pool is hard-coded to 50 workers. If Cloud Run concurrency differs or CPU throttles, you can saturate threads."

**Evidence**: The pool size is configurable. Additionally, LLM concurrency is bounded by a semaphore.

`src/agent/agents/_base.py:54`:
```python
_LLM_SEMAPHORE = asyncio.Semaphore(20)
```

The retrieval uses `asyncio.gather()` with per-strategy `asyncio.wait_for()` timeouts, not a thread pool of 50. The executor-based bridging in `tools.py` uses the default `ThreadPoolExecutor` which scales automatically.

**Verdict**: Partially false. The concurrency is bounded by semaphore (20) and per-strategy timeouts, not a hardcoded 50-thread pool.
